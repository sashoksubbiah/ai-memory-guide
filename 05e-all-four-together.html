All Four Memory Types Working Together

One user query, four stores activated simultaneously — how a real agent orchestrates them all.

On this page
The scenarioStep by stepThe assembled promptKey takeawayQuiz
Setup

The scenario

A visitor walks up to a museum agent and says:

"Tell me more about that Refik piece you mentioned earlier."

One sentence. Four memory types activated simultaneously. Here is exactly what happens.

Step by Step

How the agent handles it

1

Episodic retrieval

A vector search finds the prior conversation turn where the Refik piece was mentioned. Without episodic memory, "you mentioned earlier" would fail completely.

2

Semantic retrieval

RAG pulls the full artwork description and artist biography from the museum catalogue. This is semantic knowledge about the world, not about the user.

3

Procedural invocation

An artwork_detail tool fires to fetch any live data the catalogue lacks — current location in the gallery, current exhibition. Procedural memory knows this tool exists and when to call it.

4

Working memory assembly

System prompt + retrieved episodic facts + retrieved semantic facts + tool result + the new message all sit in the active context window. The LLM answers from there.

💡 The user notices nothing

The four-store orchestration happens in milliseconds, invisibly. The user just experiences an agent that seamlessly remembered their prior conversation and knows everything about the artwork — without them repeating a single thing.

The Assembled Prompt

What the LLM actually sees

This is approximately what gets assembled:

assembled.txt
[SYSTEM]
You are a knowledgeable museum assistant...

[EPISODIC MEMORY]
- 3 minutes ago: User expressed interest in the Refik Anadol installation "Machine Hallucinations"

[SEMANTIC MEMORY]
- "Machine Hallucinations" by Refik Anadol: large-scale AI artwork using NASA data
- Refik Anadol Studio: Istanbul-born artist working with AI and architecture
- Current exhibition: through March 2025

[TOOL RESULT: artwork_detail]
- Current location: Gallery 4B, floor 2
- Live crowd level: low (8 visitors)

[USER]
"Tell me more about that Refik piece you mentioned earlier."
The Key Insight

What this means for design

Each memory type solves a different problem:

Memory typeProblem it solves
EpisodicContinuity — remembering prior moments with this specific user
SemanticKnowledge — knowing facts about the world and the user
ProceduralCapability — knowing what actions to take and which tools to call
WorkingPresence — keeping the current thread alive and coherent
✓ The goal

A well-designed agent makes all four feel seamless. The user experiences an agent that simply knows, remembers, and can do — without noticing the four distinct stores behind the curtain.
