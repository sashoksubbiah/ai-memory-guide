The Context Window is a Moving Sidewalk

The simplest memory strategy: stuff the transcript back in every turn. It works perfectly — until it doesn't.

The Ceiling

What is a context window?

Every language model has a hard limit on how much text it can process in one call, measured in tokens.

GPT-4o

128,000 tokens ≈ 90,000 words ≈ one full novel

Claude Sonnet 3.5

200,000 tokens ≈ 140,000 words ≈ two full novels

These sound huge. But system prompts, conversation history, retrieved docs, and tool outputs fill them fast.

⚠ The ceiling problem

No matter how large the window, it has a hard limit. When you hit it, something must be dropped. The question is what, and how do you decide?

Units

Tokens, not words

Models count tokens — roughly 3–4 English characters each. The word "unbelievable" is 4 tokens. The word "I" is 1.

ContentApprox. tokens
One page of English prose~750
One page of code~1,300
100,000 tokens~133 pages of text
💡 Tip

A typical production system prompt uses 500–2,000 tokens before the user even types anything.

Strategy 1

The naive strategy: keep everything

The simplest approach: include the full transcript in every prompt. Zero extra engineering, perfect recall — as long as you stay under the limit.

✓ When this is right

For short sessions — support chats, one-off queries — this is often fine. Do not over-engineer. If conversations typically fit in a few thousand tokens, just include everything.

✗ When it breaks

Once conversations grow long, you hit the ceiling. At that point you need an eviction policy — a rule for what to drop.

Demo

FIFO eviction in action

FIFO — First In, First Out — drops the oldest message when the window fills. Keep typing to see it happen.

⚠ The FIFO problem

The oldest turns are often the most important: user name, stated goals, key constraints. Dropping them blindly destroys continuity at the worst possible moment.

Better Approaches

Beyond FIFO

1

Summarization

Every N turns, compress old turns into a short paragraph using a secondary LLM call. Fewer tokens, key facts preserved. Cost: lossy.

2

Retrieval (RAG)

Store old turns in a vector DB and fetch only relevant ones per query. No eviction problem — the store is unlimited. See Chapter 06.

3

Hierarchical paging

Three tiers: verbatim recent, summarized mid-age, vector-indexed old. The agent pages memories in/out as needed. This is how MemGPT works.

Context is not a database. No query plan, no index, no access control, no TTL, no conflict resolution.
