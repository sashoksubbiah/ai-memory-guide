FIFO Eviction and Smarter Alternatives

Why dropping the oldest message first is almost always wrong — and what to do instead.

On this page
What FIFO isWhy it failsPinningSmarter strategiesQuiz
The Algorithm

What is FIFO?

FIFO — First In, First Out — is the simplest eviction policy. When the context fills, the oldest message is removed first. Simple, stateless, zero configuration.

fifo.py
from collections import deque

class FIFOMemory:
    def __init__(self, max_tokens):
        self.max_tokens = max_tokens
        self.messages = deque()
        self.token_count = 0

    def add(self, message, tokens):
        self.messages.append((message, tokens))
        self.token_count += tokens
        while self.token_count > self.max_tokens:
            old_msg, old_tokens = self.messages.popleft()
            self.token_count -= old_tokens
The Core Flaw

Why FIFO fails

⚠ Classic failure

Turn 1: "My name is Priya and I'm building a medical tool for rural India."
Turns 2–30: UI, code, design.
Turn 31: Context fills. Turn 1 is evicted.
Turn 32: User asks about compliance. The agent has no idea who they are or what they're building.

Quick Fix

Pinning critical messages

Mark important messages as pinned. They are never evicted regardless of age.

pinned.py
class PinnedMemory:
    def add(self, message, tokens, pinned=False):
        self.messages.append({
            "text": message, "tokens": tokens, "pinned": pinned
        })
        self.token_count += tokens
        while self.token_count > self.max_tokens:
            for i, msg in enumerate(self.messages):
                if not msg["pinned"]:
                    self.messages.pop(i)
                    self.token_count -= msg["tokens"]
                    break
            else:
                break  # All remaining are pinned
💡 Practical tip

Pin the first 3–5 messages by default. They almost always contain the highest-value context: identity, goals, constraints.

Complete Solutions

Smarter strategies

1

Importance scoring

Score each message before evicting: recency, user facts, explicit markers. Evict the lowest-scored first, not the oldest.

2

Summarize-then-drop

Compress a window of old turns into a compact paragraph, then replace the whole window with that summary. Lossy, but preserves meaning.

3

RAG

Store old turns in a vector DB, fetch only relevant ones per query. No eviction problem at all. See Chapter 06.

4

Hierarchical paging

Three tiers: verbatim recent, summarized mid-age, vector-indexed old. Explicit promotion/demotion between tiers. See MemGPT.
