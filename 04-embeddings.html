Embeddings Turn Meaning Into Geometry

An embedding is a list of numbers that places text in a high-dimensional space where similar meanings cluster near each other.

On this page
What embeddings areHow they workSemantic searchEmbedding modelsDemoQuiz
Definition

What an embedding is

An embedding is a vector — a list of numbers — produced by passing text through a neural network trained to put similar meanings close together in that number-space.

Raw text

"User goes hiking on weekends"

Embedding vector

[0.023, -0.412, 0.891, 0.137, ...]
1,536 numbers (OpenAI ada-002)

💡 Key insight

Real embeddings use 768–3,072 dimensions. In that space, "hiking" and "trekking" land near each other. "Database schema" and "SQL index" land near each other. Unrelated topics are far apart.

Mechanics

How embeddings are created

1

Training

A large neural network is trained on millions of text pairs, learning to position similar texts near each other in vector space.

2

Inference

You pass any text through this frozen network. Out comes a fixed-length vector representing the meaning.

3

Indexing

Store these vectors in a vector database alongside the original text.

4

Query

Embed the query. Find the vectors closest to it. Return those texts.

embed.py
from openai import OpenAI
client = OpenAI()

def embed(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return response.data[0].embedding

vec = embed("User goes hiking on weekends")
print(len(vec))  # 1536
Semantic Search

Finding memories by meaning

Search uses cosine similarity — measuring the angle between two vectors. Similarity 1.0 means identical direction (same meaning). 0.0 means perpendicular (unrelated).

search.py
import numpy as np

def cosine_similarity(a, b):
    a, b = np.array(a), np.array(b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def semantic_search(query, memory_store, top_k=5):
    query_vec = embed(query)
    results = [(cosine_similarity(query_vec, m.vec), m) for m in memory_store]
    results.sort(reverse=True)
    return [m for _, m in results[:top_k]]
⚠ 2D vs reality

Real embedding spaces have 1,536–3,072 dimensions. The 2D diagrams you see are useful illustrations, not accurate representations. The math works identically regardless of dimension.

⚠ Lock your embedding model

Once you embed your memory store, changing models requires re-embedding everything. Decide early and build a migration path.
