Read Path Code

Putting retrieval together

read_path.py
from openai import OpenAI
import numpy as np

client = OpenAI()

# 1. Embed query
def embed(text: str) -> list[float]:
    return client.embeddings.create(
        model="text-embedding-3-small", input=text
    ).data[0].embedding

# 2. Cosine similarity
def cosine_sim(a, b) -> float:
    a, b = np.array(a), np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))

# 3. Full read path (simplified)
def read_path(query: str, user_id: str, memory_store, top_k=5):
    # Need detection
    if not needs_retrieval(query):
        return []  # skip the store entirely

    # Dense retrieval
    query_vec = embed(query)
    results = [
        (cosine_sim(query_vec, m.vector), m)
        for m in memory_store
        if m.user_id == user_id and m.status == "active"
    ]
    results.sort(reverse=True)
    return [m for _, m in results[:top_k]]

# 4. Build prompt with retrieved memories
def build_prompt(system: str, memories: list, history: list, user_msg: str) -> str:
    parts = [system]
    if memories:
        parts.append("[MEMORY]\n" + "\n".join(f"- {m.text}" for m in memories))
    parts.extend([f"{r}: {c}" for r, c in history[-10:]])
    parts.append(f"User: {user_msg}")
    return "\n\n".join(parts)
Write Path Code

Extracting and storing facts

write_path.py
import json, re

EXTRACT_PROMPT = """
Analyze this conversation turn.
Extract any facts worth storing long-term about the user.
Return ONLY JSON: {"facts": [str], "worth_saving": bool}
Turn: {turn}
"""

PII_PATTERNS = [
    r"\b\d{4}[- ]?\d{4}[- ]?\d{4}[- ]?\d{4}\b",  # credit cards
    r"\b\d{3}-\d{2}-\d{4}\b",                        # SSN
    r"[a-zA-Z0-9+_.-]+@[a-zA-Z0-9.-]+",               # email (optional)
]

def extract_facts(turn: str) -> list[str]:
    response = llm.call(EXTRACT_PROMPT.format(turn=turn))
    parsed = json.loads(response)
    return parsed.get("facts", []) if parsed.get("worth_saving") else []

def pii_check(fact: str) -> bool:
    return any(re.search(p, fact) for p in PII_PATTERNS)

def write_path(turn: str, user_id: str, memory_store):
    facts = extract_facts(turn)
    for fact in facts:
        if pii_check(fact):
            continue  # drop PII silently

        # Check for conflicts
        similar = memory_store.search(fact, user_id=user_id, top_k=3)
        for s in similar:
            if conflicts(fact, s):  # semantic contradiction check
                memory_store.update(s.id, {"status": "superseded"})

        # Write clean fact
        memory_store.upsert({
            "text": fact,
            "vector": embed(fact),
            "user_id": user_id,
            "status": "active",
        })
Governance Layer

The layer that makes it production-safe

⚠ Governance is what separates a demo from a product

The read and write paths above are functional. The governance layer is what makes them trustworthy: PII protection, conflict resolution, permission scoping, and audit logging.

1

PII filter on every write

Run regex + optionally an LLM-based PII classifier before writing any fact. Redact or drop. Log the redaction event.

2

Conflict detection before every write

Embed the new fact. Search the store for similar existing facts. If contradiction detected, mark old as superseded. Write new with pointer to old.

3

user_id filter on every read

The search function must always pass user_id as a mandatory filter. Never allow a read without scope. This is a security invariant.

4

Audit log on writes and deletes

Every upsert and deletion is logged: who wrote it, from what turn, what time, and why. This is your compliance and debugging trail.

5

Decay runner

A nightly background job reduces the confidence score of unreinforced facts over time. High-age, low-confidence facts are reviewed or removed.
