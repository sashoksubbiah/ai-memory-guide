Try It Yourself: Memory On vs Off

The fastest way to understand what memory does is to watch a conversation break without it.

On this page
Why this mattersLive demoThe codeLimitations
Motivation

Why this demo

In Chapter 01 we established that a language model is stateless. Reading about it is one thing. Experiencing it break is another.

Two modes: memory ON prepends the full transcript every turn. Memory OFF isolates each turn completely — the model sees only the current message.

🔫 Suggested experiment

Step 1: Memory ON. Say "My name is Alex". Say "I like hiking". Ask "What do you know about me?" — both facts should be recalled.
Step 2: Click Reset. Toggle memory OFF. Repeat the same three messages. Watch step 3 fail completely.

Under the Hood

What the code looks like

This is essentially one extra line: prepend the transcript. Simple in theory. But it breaks the moment the conversation grows longer than the context window.

simple_memory.py
transcript = []

def chat(user_message: str) -> str:
    # Memory ON: include full history
    prompt = "\n".join(transcript) + "\nUser: " + user_message

    # Memory OFF would just be:
    # prompt = user_message

    response = llm.call(prompt)
    transcript.append(f"User: {user_message}")
    transcript.append(f"Assistant: {response}")
    return response

Once the transcript grows past the context limit, something must be dropped. That is exactly what Chapter 02 is about.

Honesty

Limitations of this demo

⚠ Toy simulation

The AI responses use simple pattern-matching, not a real language model. The goal is to show the structural difference memory makes — not to demonstrate LLM quality.
