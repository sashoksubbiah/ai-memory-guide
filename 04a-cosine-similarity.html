Cosine Similarity: Measuring Meaning Distance

How agents find the memories most relevant to a query — the math behind semantic search.

On this page
What it isThe mathWhy angle not distanceLimitationsQuiz
Definition

What cosine similarity is

Cosine similarity measures the angle between two vectors. If they point in the same direction, similarity is 1.0. Perpendicular: 0.0. Opposite: −1.0.

Score rangeInterpretation
0.95–1.0Nearly identical meaning
0.8–0.95Highly related
0.5–0.8Somewhat related
0.0–0.5Loosely related or unrelated
Below 0Opposing meaning
The Math

How it's calculated

The formula: cos(θ) = (a ⋅ b) / (|a| × |b|) — the dot product divided by the product of magnitudes.

cosine.py
import numpy as np

def cosine_similarity(a, b):
    a_arr = np.array(a)
    b_arr = np.array(b)
    dot_product = np.dot(a_arr, b_arr)
    magnitude_a = np.linalg.norm(a_arr)
    magnitude_b = np.linalg.norm(b_arr)
    return float(dot_product / (magnitude_a * magnitude_b))

# In practice, vector databases compute this for you at scale
# using hardware-accelerated FAISS indexes
💡 In practice

You almost never implement this yourself. Vector databases like Pinecone, Weaviate, and pgvector compute it for you, often searching billions of vectors in milliseconds using FAISS.

The Intuition

Why angle not Euclidean distance

Two identical sentences might have vectors of different magnitudes if one is longer (contains more tokens). The Euclidean distance between them would be non-zero despite identical meaning. The angle between them would be zero.

Cosine similarity is magnitude-invariant: it captures the direction of meaning regardless of text length. This is why it is the standard metric for semantic similarity.
