When Writes Conflict: Memory Governance

Naive append leaks PII and produces contradictions. Governed memory handles supersession, redaction, and temporal history.

On this page
The conflict problemNaive appendNaive overwriteGoverned memoryImplementationQuiz
The Problem

When facts contradict each other

Memory is not a one-time write. Users move. Preferences change. Jobs switch. A user who told you they live in Istanbul two months ago might now be in Berlin.

How do you handle conflicting writes? Three strategies, with very different outcomes.

📥 Incoming utterances

1. "User lives in Istanbul."
2. "User is moving to Berlin next month."
3. "User's credit card: 4111-1111-1111-1111"

Strategy 1

Naive append

✗ Naive Append Result
storedUser lives in Istanbul.
storedUser is moving to Berlin next month.
⚠ PIIUser credit card: 4111-1111-1111-1111

The agent now sees two contradictory city facts and will pick one arbitrarily. PII is stored unguarded in plaintext.

Strategy 2

Naive overwrite

⚠ Naive Overwrite Result
currentUser lives in Berlin.
⚠ PIIUser credit card: 4111-1111-1111-1111

Istanbul is permanently gone. If the user asks "Where did I used to live?", the agent cannot answer. History is lost. PII still stored.

Strategy 3

Governed memory

✓ Governed Result
new · valid next monthUser lives in Berlin.
superseded · kept for temporalUser lived in Istanbul (until next month).
redacted[REDACTED — payment instrument. PII filter triggered.]

✓ No contradiction. ✓ History preserved for temporal queries. ✓ PII never touches the store.

Implementation

How to implement governance

governed_write.py
import re
from datetime import datetime

PII_PATTERNS = [r"\b\d{4}[- ]?\d{4}[- ]?\d{4}[- ]?\d{4}\b"]

def governed_write(fact: str, store):
    # 1. PII check
    if any(re.search(p, fact) for p in PII_PATTERNS):
        store.write({"text": "[REDACTED]", "type": "redacted"})
        return

    # 2. Check for conflicts
    existing = store.search_similar(fact, top_k=3)
    for e in existing:
        if conflicts(fact, e):
            store.update(e.id, {"status": "superseded"})  # mark, not delete

    # 3. Write new fact
    store.write({
        "text": fact,
        "status": "active",
        "created_at": datetime.utcnow(),
    })
