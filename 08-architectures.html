Choosing

How to pick an architecture

Start with the simplest architecture that meets your requirements. Add complexity only when you can measure that it solves a specific problem.

1

Prototype (Simple buffer)

If sessions are short and context fits, start here.

2

Scale up (Vector store)

When sessions grow long or span multiple sessions, add semantic retrieval.

3

Add structure (Knowledge graph)

When you need multi-hop reasoning or relational queries.

4

Handle complexity (MemGPT / Letta)

When you need the agent to manage its own memory context or edit its own store.

Combining Architectures

Most production systems use multiple

Most production agents combine several:

  • Vector store for semantic recall of user facts and past episodes
  • KV cache (hot tier) for the top 10 most-accessed facts per user
  • Knowledge graph for structured relationships
  • Governance layer (supersession, PII) running as a write-path service
💡 Quote

Memory governance is what separates a one-off demo from a production agent.
