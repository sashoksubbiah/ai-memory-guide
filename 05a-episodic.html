Episodic Memory: What Happened, When

Time-stamped logs of past interactions. The kind of memory that makes "remember that project from last week?" actually work.

On this page
What it isThe write pathRetrievalExampleQuiz
Definition

What episodic memory is

Episodic memory stores time-ordered logs of past events. Each entry has a when, a what, and ideally a who. Retrieval is most often by recency — "what did we just talk about?" — but you can also query by date range or topic.

📜 Real-world analogy

Episodic memory is like a diary. You do not just know facts about your friend (that is semantic). You remember specific moments with them — that time you argued about Python vs JavaScript in 2022. That is episodic.

Writing

The write path

After each conversation turn, the agent extracts a short summary and stores it with metadata:

episodic_write.py
import uuid
from datetime import datetime

def write_episode(turn: dict, vector_db):
    # 1. Extract a concise summary of this turn
    summary = llm.call(f"Summarize in one sentence: {turn['user']} / {turn['agent']}")

    # 2. Embed the summary
    vec = embed(summary)

    # 3. Store with rich metadata
    vector_db.upsert({
        "id": str(uuid.uuid4()),
        "vector": vec,
        "text": summary,
        "user_id": turn["user_id"],
        "session_id": turn["session_id"],
        "created_at": datetime.utcnow().isoformat(),
        "turn_index": turn["index"]
    })
Retrieval

Finding past episodes

Retrieval can combine semantic search (what was discussed) with temporal filters (when):

episodic_search.py
def find_past_episodes(query, user_id, days_ago=None, top_k=5):
    query_vec = embed(query)
    filters = {"user_id": user_id}

    if days_ago:
        cutoff = (datetime.utcnow() - timedelta(days=days_ago)).isoformat()
        filters["created_at"] = {"$gte": cutoff}

    return vector_db.search(
        vector=query_vec,
        filter=filters,
        top_k=top_k
    )
💡 ChatGPT's memory feature

The simplest version of episodic memory is exactly ChatGPT's memory feature: persistent recall across sessions so you do not have to re-explain yourself every time. Under the hood it is a vector store of past conversation summaries.

Example

In the real world

User: "Can you help me with that Python script we worked on last week?"

  1. Agent embeds the query.
  2. Searches episodic store filtered to user_id and last 7 days.
  3. Retrieves: "User was debugging a Pandas groupby aggregation for sales data."
  4. Injects into prompt. Agent responds with full context of what was built and where they left off.
What is the key advantage of episodic memory over semantic memory?
AEpisodic memory is always faster to retrieve
BEpisodic memory captures temporal context — when something happened, not just what
CEpisodic memory stores more data
DEpisodic memory does not require embeddings
