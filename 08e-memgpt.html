Hierarchical Memory: MemGPT and the Paging Model

The agent manages its own context: hot in-prompt, warm on disk, cold in archival. Unlimited effective context.

On this page
The problemThe paging modelMemory operationsLimitationsQuiz
Motivation

The problem MemGPT solves

Standard RAG retrieves memories based on the current query. But sometimes an agent needs to reason about its own memory context — deciding what to page in, what to page out, what to consolidate.

MemGPT (2023, Packer et al.) applied the idea of virtual memory paging to LLM context management. The agent explicitly manages what lives in its "working set" versus what is paged to disk.

The Paging Model

Three memory tiers

In-Context (hot)

Active working memory. Verbatim content in the current prompt. Immediately accessible. Limited by context window.

External (warm)

Recent episodes and facts on disk. Retrieval required (milliseconds). Good for recent user history.

Archival (cold)

Long-ago episodes and rarely-accessed data. Slower retrieval. Good for distant history and reference material.

💡 The key difference

In standard RAG, the retrieval system decides what to fetch. In MemGPT, the agent itself issues paging commands. It is self-aware about its own memory context.

Memory Operations

What the agent can do

memgpt_ops.py
# MemGPT-style memory operations the agent can call:

# Page a memory from disk into context
memory.recall(query="last conversation about the user's project")

# Archive a section of context to external storage
memory.archive(content="[old context block]", reason="no longer needed in hot tier")

# Retrieve from archival
memory.search_archival(query="early discussion about database schema")

# The agent decides when and what to page, not the retrieval system
# This is the key insight: the agent is memory-aware
Limitations

Where the paging model struggles

⚠ Complexity cost

Self-managed paging requires: tool definitions for memory ops, system prompt instructions for when to page, evaluation of paging decisions, and careful handling of paging failures. This is significantly more complex than standard RAG.
