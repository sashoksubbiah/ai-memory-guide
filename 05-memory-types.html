Four Memories, One Agent

Cognitive science has been carving up "memory" for a century. Agent frameworks borrow the same vocabulary because the distinctions are genuinely useful.

Context

Why the cognitive science taxonomy helps

Human memory researchers have long distinguished between different kinds of memory. When AI agent frameworks borrowed this taxonomy, it was not just for analogy — each type has a genuinely different shape, a different write rule, and a different retrieval strategy.

💡 The key insight

Understanding which memory type a piece of information belongs to tells you how to store it, how to retrieve it, and when to write it. These are not just labels — they drive architecture decisions.

The Four Types

Episodic, Semantic, Procedural, Working

TypeWhat it storesExampleRetrieval pattern
EpisodicWhat happened, whenUser asked about Berserk on TuesdayBy recency or date range
SemanticFacts and relationsUser is vegetarian, lives in BerlinBy semantic similarity
ProceduralSkills and toolsHow to call the weather APIBy tool name or intent classification
WorkingActive context right nowCurrent conversation turnsNone — already in the prompt
⚠ Boundaries blur in practice

Real agents rarely have four separate databases. Often a single vector store handles both episodic and semantic, while procedural memory lives in the system prompt and working memory is just the context window.
