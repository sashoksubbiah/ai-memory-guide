HyDE: Hypothetical Document Embeddings

Questions and answers are different shapes in embedding space. HyDE bridges the gap by searching with a generated fake answer.

On this page
The problemThe solutionImplementationTradeoffsQuiz
The Asymmetry Problem

Questions vs answers in embedding space

Questions and their answers have systematically different embedding shapes. "Where does the user live?" embeds near other question-shaped text. "The user lives in Berlin" embeds near fact-shaped text.

When you embed a question and search your memory store of facts, you may miss relevant memories even though they would perfectly answer the question — because the question embedding is not close to the answer embedding.

⚠ This is a real problem

The question "What hiking trails does the user enjoy?" may not retrieve the memory "User goes hiking in the canyons on Saturdays" because the question and fact have different embedding shapes, even though one directly answers the other.

The Solution

How HyDE works

1

Embed the question normally

Start with the user's raw query.

2

Generate a hypothetical answer

Ask the LLM to write what the ideal answer might look like. It does not need to be factually correct — just answer-shaped.

3

Embed the hypothetical answer

Embed this fake answer instead of the question.

4

Search with the answer embedding

Now you are searching with something that looks like what you are looking for, not like the question you are asking.

hyde.py
def hyde_retrieve(query, memory_store, top_k=5):
    # Step 1: Generate a hypothetical answer
    hyde_prompt = f"""
    Write a plausible one-paragraph answer to this question.
    The answer does not need to be factually correct.
    Question: {query}
    """
    hypothetical_answer = llm.call(hyde_prompt)

    # Step 2: Embed the hypothetical answer (not the question)
    answer_vec = embed(hypothetical_answer)

    # Step 3: Search with the answer-shaped embedding
    return memory_store.search(answer_vec, top_k=top_k)
Tradeoffs

When to use HyDE

AspectDetail
Quality gain10–20% improvement in recall on question-type queries in most benchmarks
CostOne extra LLM call per query — usually a cheap, fast model
Latency+50–200ms for the hypothetical answer generation
Failure modeIf the LLM generates a wildly wrong hypothetical, retrieval quality can drop
Best forQuestion-answer retrieval. Less useful for keyword or ID lookups
💡 When to enable HyDE

Enable HyDE when your users are asking questions about the agent's knowledge base, not when they are requesting actions. "What do I usually order for lunch?" benefits. "Set a reminder for 3pm" does not.
