Working Memory: The Current Scratchpad

The context window in active use — everything the model is reasoning over right now.

On this page
What it isWhat's in itCapacityDuring reasoningQuiz
Definition

What working memory is

Working memory is the active contents of the context window. It is everything spread out on the model's desk right now. When the session ends, it is gone.

💡 RAM analogy

Working memory is like RAM: fast, small, volatile. Long-term memory is the hard drive: slow, large, persistent. Both are necessary and neither replaces the other.

Chain of Thought

Working memory during reasoning

When a model generates a chain of thought — reasoning, planning, step-by-step problem solving — that reasoning is also working memory. The model writes thinking into the context window and can refer back to earlier reasoning steps.

This is why larger context windows improve performance on hard tasks — the model has more desk space for its work.

💡 Implication

For complex multi-step tasks, do not aggressively compress the prompt. Give the model room to reason. The cost of a slightly larger prompt is almost always worth the quality gain.
