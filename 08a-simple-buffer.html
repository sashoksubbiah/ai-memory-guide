Simple Buffer: The Starting Point

Zero infrastructure, perfect for prototypes. Not for production.

On this page
What it isImplementationLimitationsWhen to use itQuiz
Definition

What a simple buffer is

A simple buffer is the most minimal possible memory strategy: keep a list of recent conversation turns in memory and prepend them to every prompt. No database. No embeddings. No infrastructure.

💡 This is how 99% of chatbot demos work

And it is also why most demos break when you add a 20th message.

Implementation

How to build it

simple_buffer.py
class SimpleBuffer:
    def __init__(self, max_tokens=4000):
        self.messages = []  # [(role, content), ...]
        self.max_tokens = max_tokens

    def add(self, role, content):
        self.messages.append((role, content))
        # Trim to token budget (rough estimate)
        while self.token_count() > self.max_tokens and len(self.messages) > 1:
            self.messages.pop(0)  # FIFO eviction

    def token_count(self):
        return sum(len(c) // 3.5 for _, c in self.messages)

    def to_prompt(self):
        return "\n".join(f"{role}: {content}" for role, content in self.messages)
⚠ FIFO eviction rears its head

Once the buffer fills, this degrades to pure FIFO — with all its problems. See Chapter 02a.

Limitations

Why this does not scale

LimitationImpact
Context window boundCannot remember beyond N turns
FIFO evictionLoses the most important early context first
No semantic recallCannot answer "what did we discuss last week?"
No persistenceMemory dies when the process restarts
No governanceNo PII filtering, supersession, or audit trail
When to Use It

The right contexts for a simple buffer

✓ Use simple buffer when

You are building a prototype. Sessions are short (<10 turns). There is no cross-session continuity requirement. You want to ship something fast and add memory later.

✗ Do not use in production when

Users expect multi-session continuity. Conversations regularly exceed 10–20 turns. PII may appear. You need audit trails or conflict resolution.
