Need Detection: Should We Hit the Memory Store at All?

Calling retrieval is not free. Smart agents detect first whether a query even needs past context.

On this page
Why skip retrieval?Detection signalsImplementationExamplesQuiz
Cost Awareness

Why skip retrieval when possible

Every retrieval call costs: latency (50–300ms), compute (embedding + vector search), and tokens (the retrieved memories go into the prompt, consuming context budget).

If the model can answer a query cold — from training data or the current context — adding retrieval just adds noise and cost.

💡 The key question

Before running retrieval, ask: does answering this query require knowing something that would only be in the user's memory store?

Implementation

How to detect need

need_detection.py
NEED_PROMPT = """
Does answering this query require knowing facts specific to this user
or their past conversations? Answer with JSON:
{"needs_retrieval": true/false, "reason": "..."}

Query: {query}
"""

def needs_retrieval(query: str) -> bool:
    result = llm.call(NEED_PROMPT.format(query=query), model="gpt-4o-mini")
    parsed = json.loads(result)
    return parsed["needs_retrieval"]

# Fast rule-based pre-filter (before the LLM call):
QUERY_SIGNALS = ["usually", "often", "like before", "what did we", "remember when",
                 "my preference", "as I mentioned", "last time"]
def fast_needs_retrieval(query: str) -> bool:
    return any(signal in query.lower() for signal in QUERY_SIGNALS)
💡 Two-stage detection

Run the fast rule-based filter first. Only run the LLM classifier on queries that slip through. This keeps the detection overhead minimal.
