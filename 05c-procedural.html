Procedural Memory: Learned Skills and Tools

Encoded as system prompt instructions or callable tools — how agents know what they can do.

On this page
What it isHow it is encodedTool-use as proceduralUpdating itQuiz
Definition

What procedural memory is

Procedural memory is knowledge of how to do things, not knowledge of facts. A human's procedural memory includes riding a bike, touch-typing, and playing an instrument — skills encoded as automatic processes, not as retrievable facts.

💡 For AI agents

Procedural memory covers: how to call tools, how to format responses, how to handle edge cases, which APIs to use for which tasks. It is the agent's skill set.

Encoding

How procedural memory is encoded

Unlike episodic and semantic, procedural memory is almost never stored in a vector database. It lives in two places:

📄 System Prompt

Instructions, rules, persona, response format guidelines, and step-by-step processes are written into the system prompt. They are always present in working memory.

🔨 Tool Definitions

Callable functions — search, calculator, database queries — defined in the tools parameter of the LLM call. The model knows when and how to invoke them.

Tool-Use as Procedural Memory

How tools work

tools.py
tools = [
    {
        "name": "search_memories",
        "description": "Search the user's long-term memory store",
        "parameters": {
            "type": "object",
            "properties": {
                "query": {"type": "string", "description": "What to search for"},
                "top_k": {"type": "integer", "default": 5}
            },
            "required": ["query"]
        }
    },
    {
        "name": "get_weather",
        "description": "Get current weather for a location",
        "parameters": {
            "type": "object",
            "properties": {"location": {"type": "string"}},
            "required": ["location"]
        }
    }
]
💡 The model learns which tool to use

The model is trained on tool-use patterns. Given a user asking about weather, it knows to call get_weather. This is procedural knowledge — not stored, but trained into the model weights.

Updating

How to update procedural memory

Updating procedural memory is different from updating episodic or semantic. You cannot just write a new vector. You must change the system prompt or tool definitions.

1

Identify the skill

What behavior needs to change? When should a tool be called differently? What new edge case needs handling?

2

Update the system prompt

Add, modify, or remove instructions. Version control your prompts. Test the change.

3

Add or modify tools

Add new API connections or update existing tool descriptions to match new behavior.

4

Evaluate

Run evals to confirm the new behavior works correctly and did not regress existing skills.

⚠ Procedural memory is not per-user

Unlike semantic and episodic memory, procedural memory is shared across all users of the agent. Changing the system prompt affects everyone.
