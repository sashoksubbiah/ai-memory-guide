Reciprocal Rank Fusion: Merging Ranked Lists

How to combine dense, sparse, and graph retrieval results without knowing each retriever's internal scores.

On this page
Why multiple retrieversThe RRF formulaImplementationWorked exampleQuiz
The Case

Why run multiple retrievers

No single retriever is optimal for all queries:

RetrieverWins atLoses at
Dense (embeddings)Paraphrase, intent, semantic similarityExact rare tokens, IDs, codes
Sparse (BM25)Exact keyword match, unique identifiersParaphrase, intent
Graph walkMulti-hop relational reasoningUnstructured text similarity
💡 The intuition

Run all three retrievers in parallel. If a memory appears high in multiple ranked lists, it is almost certainly relevant. RRF formalizes this intuition into a score.

The Math

The RRF formula

RRF assigns each document a score based on its rank position across all retrievers:

rrf_formula.py
RRF_score(d) = sum(1 / (k + rank_i(d)) for all retrievers i)

# k = 60 (standard constant, reduces impact of very top ranks)
# rank_i(d) = rank of document d in retriever i (1-indexed)
# Documents not returned by a retriever are excluded from that sum
💡 Why k=60?

The constant k smooths the impact of top ranks. Without it, rank 1 gets 1.0 and rank 2 gets 0.5 — a huge jump. With k=60, rank 1 gets 1/61 ≈ 0.0164 and rank 2 gets 1/62 ≈ 0.0161 — a much smaller cliff. This makes the fusion more robust.

Implementation

How to implement RRF

rrf.py
def reciprocal_rank_fusion(results_lists, k=60):
    """
    results_lists: list of lists of document IDs, each in ranked order
    Returns: dict of {doc_id: rrf_score}, sorted by score
    """
    scores = {}
    for ranked_list in results_lists:
        for rank, doc_id in enumerate(ranked_list, start=1):
            if doc_id not in scores:
                scores[doc_id] = 0.0
            scores[doc_id] += 1.0 / (k + rank)

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
