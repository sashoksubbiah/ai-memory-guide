Semantic Memory: Facts and Relations

General knowledge and user facts stored for retrieval — the RAG-shaped memory most production agents rely on.

On this page
What it isStored structureThe RAG connectionExampleQuiz
Definition

What semantic memory is

Semantic memory stores facts and relationships that are not tied to a specific time or event. "User is vegetarian" is semantic. "User mentioned being vegetarian during the Tuesday onboarding session" is episodic.

💡 Real-world analogy

Semantic memory is like your general knowledge about a friend: their dietary preferences, job, hobbies, where they live. You know these things, but you do not necessarily remember the specific conversation where you learned them.

Stored Structure

What semantic entries look like

Semantic memories are typically stored as clean extracted facts:

semantic_store.py
# Bad: raw turn stored verbatim
{"text": "User said they don't eat meat or fish on Tuesday"}

# Good: extracted semantic fact
{
    "text": "User is vegetarian",
    "confidence": 0.95,
    "source": "turn_0042",
    "created_at": "2024-03-15",
    "status": "active"
}

# Even better: with entity links
{
    "subject": "user:priya_123",
    "predicate": "dietary_preference",
    "object": "vegetarian",
    "confidence": 0.95
}
⚠ Extract, don't store raw

Storing raw conversation text as semantic memory bloats the store and hurts retrieval precision. Always extract clean facts before storing.

The RAG Connection

How semantic memory powers RAG

Most RAG (Retrieval-Augmented Generation) systems are essentially semantic memory systems. The pattern is identical:

1

Embed

Convert the user query to a vector

2

Search

Find the semantically closest facts in the store

3

Inject

Prepend the retrieved facts into the prompt

4

Generate

The LLM answers with access to the relevant facts

💡 Difference from document RAG

User-facing semantic memory stores facts about the user. Document RAG stores facts from external documents. Both use the same retrieval mechanism, but the content and write patterns differ significantly.

Example

In the real world

User: "What restaurants can you recommend near me?"

  1. Agent embeds the query.
  2. Retrieves: "User lives in Highland Park, LA." "User prefers sushi." "User is vegetarian."
  3. Prompt now contains these facts.
  4. Agent recommends vegetarian-friendly sushi spots in Highland Park without asking for any of this context.
